# Android God Mode: The Ultimate Performance & Thermal Bypass Guide

This repository documents the definitive, step-by-step process to unlock the absolute maximum hardware performance on rooted Android devices (highly optimized for modern Snapdragon chipsets). 

By combining GPU driver updates, manual voltage tuning, kernel governor optimization, and a surgical, custom-built thermal daemon, this guide completely strips away OS and kernel-level throttling. It allows sustained maximum clock speeds during heavy workloads (like Genshin Impact) without triggering Binder IPC crashes or anti-cheat soft-locks.

**⚠️ EXTREME WARNING:** These modifications completely remove your device's thermal safety nets. Your device will get exceptionally hot during sustained loads. You are solely responsible for any hardware degradation, battery bloat, or melted silicon. Use active external cooling.

---

## 🛠️ Prerequisites
* **Root Access:** Magisk or KernelSU installed.
* **Unlocked Bootloader:** Required for root and modifying partitions.
* **Basic Terminal Knowledge:** Termux or ADB shell access.

---

## Phase 1: Adreno GPU Driver Update
Stock GPU drivers are often outdated and unoptimized. Updating to the latest Adreno Vulkan/OpenGL drivers massively improves shader compilation and frame pacing.
https://xdaforums.com/t/adreno-650-819v2-adreno-730-version-837-magisk-driver-update-module-cn-row-aosp-compatible.4739332/

1. Find the latest compatible Adreno GPU driver for your specific Snapdragon SoC (usually distributed as a `.zip` file in modding communities).
2. Open **Magisk**, go to the **Modules** tab, and select **Install from storage**.
3. Flash the driver `.zip` and reboot.

---

## Phase 2: KonaBess Manual Overclocking & Undervolting
We use KonaBess to unpack the GPU vendor image and manually edit the frequency tables. 

1. Install **KonaBess** and grant it root access.
2. Select **Backup Current Image** (Do not skip this).
3. Select **Unpack boot/vendor_boot**.
4. Edit the GPU frequency tables:
    * **Overclocking:** Add a new, higher maximum frequency tier. 
    * **Undervolting:** Slightly lower the voltage requests for your highest frequency tiers. This is crucial for sustained performance, as it reduces the raw heat generated by the SoC.
5. Repack and flash the modified image, then reboot.

*Note: If heavy 3D games hang on a white screen or crash instantly upon rendering, your undervolt is too aggressive. Revert to your backup and increase the voltage slightly.*

---

## Phase 3: SmartPack Kernel Optimization
Android's default CPU/GPU schedulers prioritize battery life over raw performance, leading to micro-stutters. We force the kernel to stop dropping frequencies.

1. Install **SmartPack-Kernel Manager (SPKM)** and grant root access.
2. Navigate to the **CPU** tab and change the **Governor** from `schedutil` (or default) to `performance`.
3. Navigate to the **GPU** tab and change the **GPU Governor** to `performance`.
4. Apply these settings on boot.

---

## Phase 4: Kreapic Disable Thermal Universal
To establish a baseline suppression of standard Android thermal limits, we use a community-trusted Magisk module.

1. Download the **kreapic disable thermal universal** Magisk module.
2. Flash it via the Magisk Manager Modules tab.
3. Reboot the device. 

*(While powerful, this module alone is often not enough to stop aggressive, hardware-level kernel polling on heavily modified ROMs, which brings us to Phase 5).*

## Phase 5: The Custom Kernel Thermal Daemon (The Core Bypass)
To achieve perfect stability without breaking Android's Binder IPC (which causes heavy games like Genshin Impact to hang during startup), we rely on a custom, invisible background daemon. 

Instead of freezing the userspace OS daemons (which apps need to talk to), this daemon surgically hammers the low-level Linux Kernel Thermal Zones into a disabled state every 5 seconds.

### Step 1: Create the Brain (The Daemon)
This script runs in an infinite loop, shutting down the hardware kernel zones.

Create `/data/adb/thermal_daemon.sh` and paste the following:

```bash
#!/system/bin/sh

# Wait patiently for boot to finish
while [ "$(getprop sys.boot_completed)" != "1" ]; do
    sleep 5
done
sleep 15

# Apply system property overrides
resetprop sys.thermal.enable 0
resetprop persist.sys.thermal.config 0

# Infinite silent watchdog loop
while true; do
    
    # Force-disable Linux Kernel Thermal Zones
    for zone in /sys/class/thermal/thermal_zone*/mode; do
        echo "disabled" > "$zone" 2>/dev/null
    done

    # Rest for 5 seconds before sweeping again
    sleep 5
done
```

Set the execution permissions:
```bash
chmod 755 /data/adb/thermal_daemon.sh
```

### Step 2: Create the Launcher
To prevent Magisk's boot watchdog from killing our infinite loop, we use a separate script to launch the daemon and forcefully detach it into the background.

Create `/data/adb/service.d/start_thermal.sh` and paste the following:

```bash
#!/system/bin/sh

# Launch the daemon, redirect all inputs/outputs to the void, and background it
nohup sh /data/adb/thermal_daemon.sh </dev/null >/dev/null 2>&1 &
```

Set the execution permissions:
```bash
chmod 755 /data/adb/service.d/start_thermal.sh
```

Upon your next reboot, Magisk will execute the launcher, which instantly drops the daemon into the background. 

You can verify the daemon is running silently by typing this into a root terminal:
```bash
ps -ef | grep thermal_daemon
```
